    • Barcode Writer in Pure PostScript

    Project maintained by Terry Burton Hosted on GitHub Pages

    Barcode Writer in Pure Postscript generates all barcode formats entirely within PostScript so that the process of converting the input data into the printed output can be performed by the printer or RIP itself. This is ideal for variable data printing (VDP) and avoids the need to re-implement the barcode generation process whenever your language needs change.

    It makes including any barcode within a PostScript document as simple as inserting the following directive:

    0 0 moveto (978-1-86074-271-2) (includetext)
/isbn /uk.co.terryburton.bwipp findresource exec

    BWIPP supports the following features:

    • GS1 Application Identifier syntax linting
    • GS1 Digital Link URI linting
    • Extended Channel Interpretation (ECI) support
    • Optimised encoding to ensure minimal symbol size
    • Print growth compensation for all formats, including colour
    • Vector output with minimal number of polygons
    • Dotty matrix support
    • Named colours, advanced colour spaces, spot colours and separations

    If it doesn't contain the format or feature you are looking for then take a look at the outstanding features, nudge an existing request or make a new one and check back soon.

    The resource is modular so you need only include the definitions for the symbologies that you will produce within your target document or labels, making the resource very lightweight.

    Since this resource is written in PostScript and interpreted within the virtual machine of a printer it is compatible with virtually any hardware, operating system and application platform, for example Windows 95/XP on i386, Mac OS X/Linux on Power PC, Solaris on SPARC, and OpenVMS on Alpha AXP.

    Use

    There are many ways that you can use the PostScript resources to generate barcodes.

    Simple instructions for getting started with creating and placing barcodes in your documents are provided in the quick guide.

    Besides direct inclusion within the Prolog section of a PostScript document, the resources are suitable for use as a PostScript library by Adobe Distiller, GhostScript, a printer hard disk or a document manager using the standard "named resources" mechanism.

    With a PostScript printer an alternative approach to embedding the resources within each document or saving them on the printer's hard drive is to install the BWIPP named resources within your printer's initial job virtual machine. Doing this will give the device barcode production capabilities for all subsequent jobs until it is reset. In effect this converts a commodity laser printer into an extremely capable barcode printer all through the use of device-resident software, no font chips required.

    More detail is provided in the download section below.

    Download

    The current release is available for download in a number of flavours supporting different use cases from here.

    BWIPP is essentially a set of generic PostScript Level 2 named resources that are provided in four flavours for ease of use. The flavour that you will find most useful depends on how you intend to deploy the library.

    "Packaged" or "unpackaged": The named resources have been packaged for DSC conformance, portability and ease of distribution. You will most likely want to use a packaged flavour in production, however the unpackaged versions of the resources are useful for understanding the PostScript code, developing the library and debugging. They are functionally equivalent an can be used interchangably.

    "Separate files" or "monolithic": The resource is provided as separate files that are formatted for direct use by Adobe Distiller, GhostScript, a printer hard disk or a document manager. The monolithic flavours contain all of the resources in a single file that is suitable for inclusion in the Prolog section of a PostScript document template or installing to a printer's initial job VM.

    We provide instructions for the seperate files and monolithic use cases.

    This leads to the following set of four files.

    For production use:

    • postscriptbarcode-packaged-resource – Packaged; Separate files.
    • postscriptbarcode-monolithic-package – Packaged; Monolithic file.

    For BWIPP development:

    • postscriptbarcode-resource – Unpackaged; Seperate files.
    • postscriptbarcode-monolithic – Unpackaged; Monolithic file.

    Browse or checkout the source code from the repository or view the changelog.

    Front Ends

    Many applications embed or plugin this PostScript resource thereby making available to their users a complete or partial implementation of the BWIPP's features. The following is a list of known open source applications that make BWIPP barcodes available to their intended audience without the need to manipulate direct PostScript:

    • Web-based generator that is useful for feature testing or generating a small number of barcodes.
    • Scribus is a desktop publishing (DTP) application for Linux and Windows users.
    • SAP support is provided in the form of an output device type accessible from ABAP, SAPscript or Smart Forms.
    • pst-barcode is part of the PSTricks package for the LaTeX document publishing system.
    • Treepoem is a Python wrapper around BWIPP.
    • bwip-js is a cross-compile of BWIPP to JavaScript that can run on Node.js or in browser.
    • Eloquence is an environment for business applications whose eqpcl component provides BWIPP integration.
